Adds the given feature to the solver.

That is, a new variable is created in the solver and all its constraints are included. The second argument indicates whether the 'optional' constraint should be considered or not – note that this is needed since alternative features are modelled as not optional, but are neither optional nor mandatory.

virtual Result<SolverErrorCode, uint64_t> vara::solver::Solver::getNumberValidConfigurations ( )
pure virtual

Returns the number of valid configurations of the current constraint system (i.e., its features and its constraints).

In principle, this is a #SAT call (i.e., enumerating all configurations).

Returns
an error if the number of valid configurations can not be retried. This can be the case if there are still constraints left that were not included into the solver because of missing variables.
